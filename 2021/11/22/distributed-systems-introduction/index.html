分布式系统：绪论

Nautilus
2021-11-22
course notes › distributed systems
distributed systems

Introduction

分布式系统：多个独立的机器以单个系统的形式呈现给用户

A distributed system is a collection of independent computers that appears to its users as a single coherent system.

最大的难点：组件间如何协作

分布式系统通常被组织为中间件（middleware），其上是用户的应用，其下是机器与其间的通信网络（机器可以是异构的）

Goals of distributed system

分布式系统需要实现四个重要的目标

  • Resource accessible：使系统中的资源可用，为用户提供一个有组织且高效的方式访问系统中的资源
  • transparency：隐藏分布式细节，具体可细分为
    • access：数据的展示与资源的访问是分离的
    • location：隐藏资源的位置，如提供 URL 以访问资源
    • migration：移动资源的位置不会影响使用
    • relocation：使用中也可以移动资源
    • replication：资源可以被复制以提升可用性
    • concurrency：资源可以被多个用户并发访问（竞争性地）
    • failure：隐藏系统中的失效和恢复
  • openness：系统应当提供清晰易用的接口（至少需要屏蔽底层实现细节）
  • scalability：系统应当易于扩展
    • size：扩展系统的资源与使用系统的用户
    • geographically：地理意义上的扩展
    • administratively：管理性上也要易于扩展

Types of distributed system

分布式计算系统（distributed computing system）主要用于高性能计算，一般可分为 cluster 与 grid。cluster 通常由一组同构的机器使用 lan 连接而成，而 grid 底层通常是异构的，节点间的距离也会更远，通常通过 wan 连接。

云可以看作是一种分布式计算系统，根据提供服务层级的不同也可以分为 SaaS，PaaS，IaaS 等

分布式信息系统（distributed information system）用于取代传统的信息系统，一个现实中的应用是分布式的事务处理系统（需要满足 ACID）

分布式普适系统（distributed pervasive system）是粒度更小的系统，如传感器网络等。主要是提供更灵活的分布式方案
